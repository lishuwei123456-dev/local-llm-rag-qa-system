# RAG 知识库示例

RAG（Retrieval-Augmented Generation，检索增强生成）是一种把外部知识检索和大语言模型生成结合起来的问答方法。它通常先根据用户问题从知识库中检索相关片段，再把这些片段作为上下文注入 Prompt，最后由大语言模型生成回答。

RAG 的核心流程包括文档加载、文本清洗、文本切分、Embedding 向量化、向量库构建、相似度检索、Prompt 拼接和答案生成。相比只依赖模型参数知识的普通问答，RAG 可以让答案更贴近指定资料，并提供来源片段，便于用户追溯依据。

在本地大模型场景中，可以使用 Ollama 部署 Qwen 模型，通过 OpenAI-compatible API 提供模型调用能力。知识库向量化可以使用 HuggingFace Embeddings，向量索引可以使用 FAISS。前端交互可以使用 Gradio 快速搭建。

RAG 可以减少模型幻觉，但不能完全消除幻觉。工程上仍然需要控制检索质量、切分粒度、Prompt 约束、答案引用和无法回答时的兜底策略。
